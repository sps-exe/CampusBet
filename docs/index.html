Project overview

What the prototype solves

CampusArena turns informal campus game challenges into structured match rooms, tournaments, wallet activity, and rankings. It is designed as a classroom-friendly, frontend-first prototype.

01

Campus-only community

Signup is gated by institutional email patterns so the prototype is scoped to students from a college ecosystem.

02

Virtual credit economy

Users stake campus credits for skill matches. Credits are a score unit only and are not withdrawable or exchangeable.

03

Competitive loop

Create a lobby, join a match, submit results, update stats, and reflect performance across wallet, profile, and leaderboard views.

Frontend-first architecture

React pages talk to Zustand stores, stores talk to Supabase.

The project intentionally avoids a custom Express backend. Supabase handles auth and database storage, while the React SPA owns page flow and UI state.

Feature map

The prototype covers the full student competition journey.

🎮

Lobbies

Create 1v1, 2v2, squad, or free-for-all match rooms with game, title, bid amount, schedule, and rules.

🏆

Tournaments

Host or browse single-elimination style events with entry fee, prize pool, dates, participants, and rules.

📊

Leaderboard

Ranks players using completed match data and filters rankings by all time, month, or week.

💳

Wallet

Builds a transaction-style view from match outcomes so wins and losses are reflected as credit changes.

👤

Profile

Shows identity, college, lifetime stats, and match history for the active student account.

🔐

Authentication

Uses Supabase auth plus a profile table, persisted in Zustand for refresh-friendly sessions.

Data model

The schema mirrors the product workflow.

The app uses a small relational model: users become profiles, profiles create lobbies and tournaments, join tables connect players to events, and completed matches power stats.

profiles

Student identity, college, credits, role, and lifetime stats.

lobbies

Game, host, bid amount, status, schedule, description, and winner.

lobby_players

Join table that records which users entered each match room.

tournaments

Hosted events with format, entry fee, prize pool, rules, and dates.

participants

Join table between users and tournaments.

Future scope

From prototype to production-ready campus platform.

Real-time lobby updates

Use Supabase realtime channels so lobby slots, statuses, and tournament registrations update live.

Dispute and proof system

Add screenshot proof upload, host moderation, dispute states, and admin review workflows.

Campus administration

Role-based dashboards for college admins to manage games, reports, users, and suspicious activity.

Bracket automation

Generate and advance brackets automatically after each tournament round.

Notifications

Email or in-app alerts for upcoming matches, result submissions, tournament starts, and leaderboard changes.

Mobile-first PWA

Installable app shell, offline-friendly views, push notifications, and improved touch interactions.

Closing summary

CampusArena proves the full loop: challenge, compete, score, rank.

The prototype demonstrates a practical frontend-first architecture, a safe virtual-credit game economy, and a complete set of student-facing workflows suitable for a capstone demo.
